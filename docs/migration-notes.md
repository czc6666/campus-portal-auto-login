# Migration Notes

这个公开仓不是把原始工作目录原样上传，而是从一个长期私有维护的校园网自动登录项目中提炼出来的。

提炼原则：

1. **保留可复用核心**
   - `核心_core/`
   - `SchoolProfile` 类型定义
   - Collector 采集器

2. **删除高风险私有材料**
   - 真实客户入口
   - 历史故障归档
   - 历史采集包
   - 构建产物
   - 老交付脚本堆

3. **尽量不动已验证核心逻辑**
   - 当前公开版优先做仓库层迁移
   - 不以“代码重写”为主

4. **新增公开友好的入口与文档**
   - `examples/run_with_env.py`
   - `examples/.env.example`
   - `docs/*.md`

---

## 关于运行路线的补充说明

这个项目在历史上并不是只有一种交付形态。

### 早期路线
- 更偏源码运行
- 可以直接用 Playwright 自带浏览器
- 更通用，也更容易在 Linux 环境里跑
- 但通常要求用户自己有 Python 环境和依赖

### 后期路线
- 更偏 Windows 客户交付
- 用 Playwright 的包去调用系统 Edge
- 更适合配合 PyInstaller 打成单文件 exe
- 更方便直接发给客户电脑使用

这两条路不是互相否定，而是面向不同目标：

- **源码运行 / 调试 / 跨平台研究** → 倾向早期路线
- **Windows 单文件交付** → 倾向后期路线

因此公开仓在文档里会同时承认这两条路线的存在，而不会把项目描述成“只支持某一种浏览器模式”。

如果你也有一个“私有交付工作台”想转成公开仓，推荐优先借鉴这种迁移方式：

**先拆出核心骨架，再补示例和文档，而不是直接把工作目录公开。**
